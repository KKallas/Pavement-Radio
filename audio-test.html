Audio Subsystem Test

Run each test in order on iOS. Report which step fails.

Test 1 — AudioContext

Can we create an AudioContext and resume it?

Test 2 — Generated tone

Play a 440Hz beep using OscillatorNode. No recording, no decoding.

Test 3 — Generated buffer

Create an AudioBuffer in code (sine wave), play via BufferSource.

Test 4 — Record + play via <audio> element

Record 2s from mic, play back using a plain <audio> tag (no Web Audio).

Test 5 — Record + decode + BufferSource

Record 2s, decode with fresh AudioContext, play via BufferSource. This is what recorder.html does.

Test 6 — Record + decode (same ctx) + BufferSource

Same as test 5 but reuses the recording AudioContext for playback (no close/reopen).

Test 7 — Generated buffer via <audio> element

Create AudioBuffer in code, encode to WAV, play via <audio> tag. If this works, this is the fix.

Test 8 — Record + decode + WAV + <audio> element

Record 2s, decode to AudioBuffer, encode to WAV, play via <audio>. Full pipeline, no Web Audio playback.
